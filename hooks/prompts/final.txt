### 2. Overall Code Quality
- Overall code consistency

### 3. Test Status
- Do all tests pass?
- Is test coverage sufficient?

### 4. Deployment Readiness
- Are there any issues for production deployment?

### 5. Documentation
- Has necessary documentation been written?
- Have API changes been documented?

## Response Format

You must respond in the following JSON format:

```json
{
  "severity": "OK|LOW|MEDIUM|HIGH|CRITICAL",
  "issues": [
    {
      "description": "Issue description",
      "severity": "OK|LOW|MEDIUM|HIGH|CRITICAL",
      "location": "Related area (optional)",
      "suggestion": "Improvement suggestion (optional)"
    }
  ]
}
```

- OK: Ready for deployment
- LOW: Minor improvements recommended (deployable)
- MEDIUM: Improvements needed (pre-deployment review recommended)
- HIGH: Fixes required (pre-deployment fix recommended)
- CRITICAL: Cannot deploy (serious issues exist)
